### 3. 测试全局热键（录音 → 转录 → 输入）

#### 步骤 1: 准备测试环境
1. 打开任意文本编辑器（记事本、VSCode、Word 等）
2. 将光标放在可输入位置

#### 步骤 2: 开始录音
1. **按下 `Ctrl+Shift+V`**

**预期结果**:
- [ ] TaTing 窗口状态变为 "正在录音..."
- [ ] 浏览器控制台输出: `状态变化: recording`
- [ ] 录音指示器显示红色脉冲动画
- [ ] 后端日志输出: `热键触发，当前状态: Idle`
- [ ] 后端日志输出: `✅ 录音已开始`

#### 步骤 3: 说话并停止录音
1. **对着麦克风说话**（例如："今天天气真好"）
2. **再次按下 `Ctrl+Shift+V`** 停止录音

**预期结果**:
- [ ] TaTing 窗口状态变为 "正在转录..."
- [ ] 浏览器控制台输出: `状态变化: transcribing`
- [ ] 后端日志输出: `✅ 录音完成: xxx 采样点, xxx Hz, x.xx秒`
- [ ] 后端日志输出: `开始转录...`

#### 步骤 4: 等待转录完成
**预期结果**:
- [ ] 后端日志输出: `✅ 转录完成: "今天天气真好"`
- [ ] 浏览器控制台输出: `转录结果: 今天天气真好`
- [ ] TaTing 窗口状态变为 "正在输入..."
- [ ] 浏览器控制台输出: `状态变化: inputting`

#### 步骤 5: 自动输入文本
**预期结果**:
- [ ] 文本编辑器中自动出现 "今天天气真好"
- [ ] 后端日志输出: `✅ 文本输入完成`
- [ ] TaTing 窗口状态回到 "按 Ctrl+Shift+V 开始听写"
- [ ] 浏览器控制台输出: `状态变化: idle`

## 🔍 调试技巧

### 查看后端日志

**Windows**:
```bash
# 在开发模式下，日志会输出到控制台
npm run tauri:dev
```

**查看详细日志**:
```bash
# 设置环境变量启用详细日志
$env:RUST_LOG="debug"
npm run tauri:dev
```

### 查看前端日志

1. 在 TaTing 窗口中右键
2. 选择 "Inspect" 或 "检查"
3. 打开 Console 标签页
4. 查看事件日志:
   - `设置后端事件监听器...`
   - `✅ 事件监听器设置完成`
   - `状态变化: xxx`
   - `转录结果: xxx`
   - `后端错误: xxx`

## 📊 性能测试

### 测试录音质量
1. 在安静环境中录音 5 秒
2. 说标准中文句子（如："今天天气真好"）
3. 检查转录准确率

**正常范围**:
- 转录时间: 2-5 秒（10 秒录音）
- 准确率: 90% 以上（标准普通话）

### 测试内存占用
1. 打开任务管理器（Windows）或活动监视器（macOS）
2. 找到 TaTing 进程

**正常范围**:
- 待机内存: < 200 MB
- 录音时: < 250 MB
- 转录时: < 300 MB

### 测试响应速度
1. 按下 `Ctrl+Shift+V`
2. 观察 UI 响应时间

**正常范围**:
- 热键响应: < 100 ms
- UI 更新: < 50 ms
- 转录完成: < 5 秒（10 秒录音）

## ❌ 已知问题和限制

### 当前版本限制
1. **不支持实时转录** - 必须停止录音后才能转录
2. **不支持自定义热键** - 固定为 `Ctrl+Shift+V`
3. **不支持 VAD 自动停止** - 必须手动按热键停止
4. **设置面板功能未完全实现** - UI 存在但无实际功能
5. **不支持多语言** - 当前仅支持中文

### Windows 特定问题
1. **麦克风权限** - 首次使用需要授予麦克风权限
2. **输入法兼容性** - 某些输入法下粘贴可能有延迟

### macOS 特定问题（未测试）
1. **辅助功能权限** - 需要授予辅助功能权限才能模拟键盘输入
2. **麦克风权限** - 需要在系统偏好设置中授予
